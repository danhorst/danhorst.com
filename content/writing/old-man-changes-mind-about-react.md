I've had a bad attitude about [ReactJS][1] for years.
Things along the lines of:

- React solves Facebook's engineering problems, not yours
- React reinvents the wheel and in the process:
  - Throws out progressive enhancement
  - Breaks accessibility
  - Bypasses built-in browser features
  - Undermines how CSS works
- React is just a library, you have to build the rest of the framework yourself
- Modern JavaScript relies on an ever-changing quicksand of compilation and transpilation tools

I've changed my attitude.
Here's why:

- React is designed for responsive _interfaces_ not web pages (although [these words of wisdom][2] still apply)
  - I have a several years of experience building JavaScript enhancements to HTML pages for "reactive" interfaces and it isn't pretty
- Modern JavaScript (and TypeScript even more so) has tamed the weird wild wonderland that was writing JavaScript 15 years ago
- The market has spoken—React is _everywhere_ and the tooling keeps getting better

[2]: https://twitter.com/wilto/status/372080088898367488
